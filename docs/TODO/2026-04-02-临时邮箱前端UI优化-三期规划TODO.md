## 背景

当前临时邮箱前端（`temp_emails.js` + 对应 `index.html` 区块）采用简单列表展示，随着临时邮箱数量增多、提供商多样化，现有 UI 在可读性和可操作性上存在提升空间。

---

## 规划目标

### 1. 按邮箱域名分组展示

- 临时邮箱列表按 **域名（`@` 后的部分）** 自动归组
- 每组显示组名（域名）+ 该组邮箱数量
- 支持折叠/展开（默认展开）
- 对应后端无需改动，纯前端分组逻辑

**示例分组**：
```
📁 example.com (3)
  ├── abc123@example.com
  ├── xyz456@example.com
  └── def789@example.com

📁 example.com (1)
  └── test001@example.com
```

### 2. 临时邮箱整体 UI 优化

- 列表布局改进：卡片式或紧凑表格式，突出邮箱地址和创建时间
- 邮件预览内嵌展示（点击展开，无需跳转）
- 验证码提取结果高亮显示（大字号 + 复制按钮）
- 邮件状态标记（未读 / 已读）
- 搜索/过滤框（按邮箱地址或主题过滤）
- 批量操作（批量删除临时邮箱 / 批量清空邮件）

### 3. 提供商标识

- 在每个临时邮箱旁显示来源 Provider 标识（如 CF Worker 图标或 badge）
- 方便用户区分不同平台创建的临时邮箱

## 开发注意事项

- 分组逻辑应为**纯前端计算**，不引入新的后端接口
- 与简洁模式轮询 (`poll-engine.js`) 不应产生交互影响
- 保持现有邮件刷新 / 验证码提取 API 调用路径不变
